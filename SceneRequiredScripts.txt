Scene Required Scripts
======================

Last updated: Validation orchestrator now logs each issue separately and keeps a compact summary line

FSM runtime scripts (required):
- Assets/FSM/UnitBrain.cs
  - Uses `MachineDefinition` / `MachineDefinitionId` instead of direct transition list wiring.
  - Delegates state/transition lookup and eligibility checks to `StateMachineRuntime`.
  - Keeps hierarchical LCA `OnExit`/`OnEnter` traversal using runtime ancestry.
  - Exposes migration glue via `LegacyStateMappings` + `LegacyInitialState`.
- Assets/Scripts/FSM/Runtime/StateMachineRuntime.cs
  - Loads one `StateMachineDefinition` for each brain.
  - Builds runtime dictionaries for states (`stateId -> BehaviourState`) and transitions (`fromStateId -> transition list`).
  - Evaluates transition eligibility from event + condition data (`eventName`, `conditionId`, legacy `conditionDescription` tokens).
  - Resolves hierarchical parent relationships from `StateDefinition.parentStateId` with fallback to `BehaviourState.Parent`.
- Assets/Scripts/Definitions/StateMachineDefinition.cs
  - Defines machine-level `initialStateId`, state entries, and transition entries (`fromStateId`, `toStateId`, `eventName`, `conditionId`).

FSM editor automation scripts (required for authoring workflow):
- Assets/Scripts/Editor/FSM/StateMachineTransitionGenerator.cs
  - Adds menu action `Tools/FSM/Generate Transition Assets from Definitions`.
  - Generates deterministic transition assets under `Assets/FSM/Transitions/<machineId>/...`.
  - Uses stable keys (machine + from + to + event/condition/payload token) for idempotent reruns.
  - Deletes stale auto-generated transition assets absent from current definition graph.
  - Applies `FSM-AutoGeneratedTransition` label and generated metadata fields for clear separation from manual assets.
  - Includes asset import/move/delete trigger for `StateMachineDefinition` synchronization.
- Assets/FSM/BehaviourTransition.cs
  - Stores generated metadata (`isAutoGenerated`, stable key, machine/from/to IDs, condition/payload, generated comment).
  - Retains obsolete runtime role; metadata is editor tooling support only.

FSM content requirements in scenes/prefabs:
- Assign `UnitBrain.MachineDefinition` (or `MachineDefinitionId`) for each unit brain instance.
- Populate `UnitBrain.LegacyStateMappings` during migration so each definition `stateId` maps to a concrete `BehaviourState` asset.
- Ensure corresponding `StateDefinition` assets include `parentStateId` where hierarchical ancestry is required.


Validation pipeline scripts (required):
- Assets/Scripts/Validation/DefinitionValidationOrchestrator.cs
  - `RunValidationAndLog()` logs each `ValidationIssue` individually using severity-specific Unity console APIs (`Debug.LogError`, `Debug.LogWarning`, `Debug.Log`).
  - Per-issue log formatting includes code + registry + message and appends asset metadata (`AssetPath`, `AssetId`, `Field`) when available.
  - Emits a final compact summary line (`N issue(s) / M error(s)`) after detailed issue output.
- Assets/Scripts/Editor/Validation/DefinitionValidationMenu.cs
  - `ValidateAllDefinitionsInEditor` and `ValidateAllForCI` still gate behavior through `report.HasErrors` and `report.ErrorCount`.
