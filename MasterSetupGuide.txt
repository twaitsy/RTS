Master Setup Guide
==================

Last updated: UnitBrain now runs a definition-driven FSM runtime adapter via StateMachineDefinition assets

FSM event flow semantics (authoritative):

Definition-driven FSM runtime (authoritative):
- `UnitBrain` now boots `StateMachineRuntime` and resolves a machine from:
  1) directly assigned `MachineDefinition`, or
  2) `MachineDefinitionId` through `StateMachineRegistry`.
- Runtime state lookup is keyed by `StateDefinition` IDs declared in `StateMachineDefinition.States`.
- Runtime transition lookup is keyed by `fromStateId` and evaluates transition eligibility using:
  - `eventName` / legacy `conditionDescription` event token,
  - optional `conditionId` (resolved through `ConditionRegistry`),
  - optional `minElapsed` token from legacy description parsing.
- `StateMachineDefinition.InitialStateId` is preferred for start state selection.

FSM hierarchical transition lifecycle semantics:
- `UnitBrain` still computes old/new ancestry chains and performs LCA enter/exit traversal.
- Parent ancestry is now sourced from definition data when available:
  - Runtime checks `StateDefinition.parentStateId` first.
  - If no definition parent mapping exists, runtime falls back to legacy `BehaviourState.Parent`.
- During transition, `OnExit` is called from old leaf upward (excluding LCA), then `OnEnter` from below LCA down to new leaf.

Migration and deprecation notes:
- `UnitBrain.LegacyStateMappings` maps legacy `BehaviourState` assets to `StateDefinition` IDs while content migrates.
- `UnitBrain.LegacyInitialState` is a temporary fallback if no initial state ID can be resolved.
- Legacy direct wiring fields (`InitialState`, `Transitions`) remain only as deprecated compatibility fallback.
- `BehaviourTransition` assets are now marked obsolete and are no longer part of the authoritative runtime path.

Legacy validation architecture notes:
- Unity menu: `Tools/Validation/Validate All Definitions`
- Runtime bootstrap, editor menu validation, and CI use the same validation orchestration core.
- CI execute method: `DefinitionValidationMenu.ValidateAllForCI`.
